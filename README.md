# JuThesis
JuThesis - это библиотека для оптимизации тестового покрытия с учетом ограничений по времени выполнения.
Основная задача, которую решает библиотека, заключается в выборе подмножества тестов, максимизирующего покрытие измененных функций и методов при заданном бюджете времени на тестирование.

Библиотека использует модифицированный жадный алгоритм с перебором малых начальных покрытий. 
JuThesis спроектирована как языконезависимая система, работающая с унифицированным протоколом входных и выходных данных в формате JSON.

## Установка
Для установки библиотеки используйте Poetry:
```bash
poetry add juthesis
```
Или установите через pip из репозитория:
```bash
pip install git+https://github.com/vplotnikov16/JuThesis.git
```

## Быстрый старт
Создайте входной файл `input.json` (см. JSON Schema [ProtocolInput_v1.0.0.json](JuThesis/protocols/schemas/ProtocolInput_v1.0.0.json), пример по пути [examples/input.json](examples/input.json)).
Запустите решатель через командную строку:
```bash
juthesis solve input.json -o output.json
```
Результат будет записан в `output.json` (см. JSON Schema [ProtocolOutput_v1.0.0.json](JuThesis/protocols/schemas/ProtocolOutput_v1.0.0.json))

## Протоколы версии 1.0.0
JuThesis работает с унифицированным протоколом обмена данными в формате JSON.
Протокол определяет структуру входных и выходных файлов, обеспечивая независимость от языка программирования и системы тестирования.
Детальное описание протокола доступно в документе [Protocol_v1.0.0.md](docs/Protocol_v1.0.0.md).

### Входной формат
Входной файл содержит информацию о версии протокола, списке измененных функций, доступных тестах с их временем выполнения и покрытия, а также параметры решателя.
Каждая функция и каждый тест идентифицируются уникальной строкой, формат которой определяется пользователем или плагином.

Поле `modified_functions` представляет собой массив строк с уникальными идентификаторами функций.
Поле `available_tests` является словарем, где ключи - это имена тестов, а значения содержат время выполнения в секундах и список покрываемых функций.
Параметры `time_budget` и `max_initial_coverage_size` управляют работой алгоритма оптимизации.

### Выходной формат
Выходной файл содержит результат работы решателя.

Поле `tests` представляет собой массив выбранных тестов, где для каждого теста указаны его имя и список фактически покрываемых функций из множества измененных.
Поля `total_execution_time` и `total_functions_covered` содержат суммарное время выполнения выбранных тестов и общее количество покрытых функций соответственно.

## Использование CLI
JuThesis предоставляет интерфейс командной строки для решения задач оптимизации тестового покрытия.

### Команда solve
Основная команда для решения задачи оптимизации:
```bash
juthesis solve INPUT_FILE -o OUTPUT_FILE
```
Команда читает входной файл, запускает алгоритм оптимизации и записывает результат в выходной файл.
По умолчанию используется формат JSON (единственный доступный на данный момент).
В процессе выполнения в консоль выводится информация о результатах.

Пример:
```bash
juthesis solve examples/input.json -o examples/output.json
```
Вывод:
```
Selected tests:     2
Covered functions:  4
Total time:         5.0
  - test1: ['func1', 'func2']
  - test3: ['func4', 'func5']
Result saved to: examples\output.json
```

### Команда validate
Проверка корректности входного файла без запуска решателя:
```bash
juthesis validate INPUT_FILE
```
Команда выполняет валидацию структуры и содержимого входного файла, сообщая о найденных ошибках или подтверждая корректность данных.

### Команда version
Отображение версии JuThesis и поддерживаемых версий протокола.
```bash
juthesis version
```

## Использование Python API
JuThesis можно использовать непосредственно из кода Python.
В директории [examples](/examples) находятся примеры использования JuThesis.

### Работа через адаптеры
Наиболее простой способ использования библиотеки - через интерфейс адаптеров:
```python
from JuThesis.adapters import solve_with_protocol
from JuThesis.io.readers.json_reader import JsonReader
from JuThesis.io.writers.json_writer import JsonWriter

# Чтение входного файла
inp = JsonReader.read('examples/input.json')
# Решение задачи
out = solve_with_protocol(inp)
# Запись результата
JsonWriter.write(out, "examples/output.json")
```

### Работа с моделями напрямую
Для более гибкого управления можно создавать модели протокола программно:
```python
from JuThesis.adapters import solve_with_protocol
from JuThesis.protocols.models import ProtocolInput, TestInfo

# Создаем входные данные
protocol_input = ProtocolInput(
    modified_functions=["func1", "func2", "func3", "func4", "func5", "func6"],
    available_tests={
        "test1": TestInfo(
            time=3.0,
            covered_functions=["func1", "func2", "funcA"],
        ),
        "test2": TestInfo(
            time=4.0,
            covered_functions=["func2", "func3", "func4", "funcB"]
        ),
        "test3": TestInfo(
            time=2.0,
            covered_functions=["func4", "func5", "funcC"]
        ),
        "test4": TestInfo(
            time=3.0,
            covered_functions=["func3", "func6", "funcD"]
        ),
    },
    time_budget=7.0,
    max_initial_coverage_size=2
)
# Решаем задачу
result = solve_with_protocol(protocol_input)
# Выводим результат
print(f"Selected tests:     {len(result.tests)}")
print(f"Covered functions:  {result.total_functions_covered}")
print(f"Total time:         {result.total_execution_time}")
for test_info in result.tests:
    print(f"  - {test_info.test}: {test_info.functions}")
```

### Использование ядра напрямую
Для максимального контроля можно работать непосредственно с решателем:
```python
from JuThesis.core import solver
from JuThesis.core.models import SolverInput

inp = SolverInput(
    # Конечное множество функций/методов, которые требуется покрыть тестированием
    J={1, 2, 3, 4, 5, 6},
    # Конечное множество доступных тестов
    I={1, 2, 3, 4},
    # Покрытия тестов
    J_i={
        1: {1, 2},
        2: {2, 3, 4},
        3: {4, 5},
        4: {3, 6},
    },
    # Время выполнения тестов
    t_i={
        1: 3,
        2: 4,
        3: 2,
        4: 3,
    },
    # Бюджет времени тестирования
    T=7,
    # Максимальная мощность стартового набора тестов H
    K=2
)
out = solver.solve(inp)
print(out)
```

## Архитектура
JuThesis построена по модульному принципу.
Ядро библиотеки реализует алгоритм оптимизации и работает с математическими моделями задачи. 
Слой протоколов определяет формат обмена данными и обеспечивает версионирование.
Адаптеры выполняют преобразование между протокольными моделями и моделями ядра, сохраняя необходимые маппинги для обратного преобразования.
Модуль ввода-вывода предоставляет абстракции для чтения и записи данных в различных форматах. Текущая реализация поддерживает JSON.

## Разработка плагинов
JuThesis спроектирована для расширения через систему плагинов.
Плагин для конкретного языка программирования или системы тестирования должен уметь собирать информацию об измененных функциях в проекте, анализировать покрытие тестов и измерять время их выполнения, формируя входной файл в формате протокола JuThesis.
Протокол обмена данными не зависит от деталей реализации плагина.
